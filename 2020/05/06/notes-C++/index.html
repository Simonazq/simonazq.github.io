C++ Notes1 - 侯捷

Posted on 2020-05-06 | In C++
**侯捷老师的C++笔记**

不知道这是第几次说要学C++了。。。。希望这次真的学下来吧。。。。

  • Object Based - 基于对象, 面对单一class的设计
  • Object Oriented - 面向对象, 面对多重classes的设计, 类与类之间的关系

p2. 头文件与类的声明

  • 防止重复定义文件:

    1. Header中的防御式声明： 宏定义名, 防止同一个文件被包含多次，导致重复定义(redefinition), 这种方法可以被跨平台使用.
      C++ (e.g. complex.h)

    2. #program once: 保证同一个文件只被编译一次, 但当文件不同名相同内容时还是会出错.

p3. 构造函数 && inline函数

  1. inline函数: 速度更快

    • 在类内部定义的函数 or 在类外定义有inline关键字的函数(放在函数定义处而不是声明处)。
    • 但不是所有的都可以成为真正的inline函数, 由编译器来决定。(不包含while, switch, recursive)

  2. 构造函数:

    • 定义方法:
      • 声明时直接初始化: 更快更高效
      • 在函数内部赋值: 相对较慢
    • 函数重载(overloading): 在编译器中函数名其实是不同名的
      • example 1: 构造函数 - wrong
      • example 2: 函数 - correct
    • private构造函数: singleton (单例设计模式)

p4. 参数传递与返回值

  • const 常量: 不改变数据的内容
    • 常量成员函数 (const member functions)： 放在()后， 前{
  • 参数传递:
    • by value: 传值，不可以改变外部的值, 一般基本类型可以传值
    • by reference(to const): 传引用，速度更快, 可以改变值
      • 如果传入的值不改变, 加const, 否则不加
      • 返回值也尽量用reference
  • 友元函数: 自由取得friend的private值， 打破了封装
    • friend 关键字: 直接取值，不通过函数调用效率更高
    • 同一个类的实体互为友元函数

设计类的思路:
1. 数据都在private里面
2. 参数与返回值尽量以reference来传
3. const能加则加
4. 构造函数尽量使用initialization list来定义
5. 返回临时变量不能by reference, 在函数结束就消失了
